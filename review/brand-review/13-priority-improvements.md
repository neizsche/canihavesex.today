# Brand Review: Priority Improvements

This document ranks the brand and positioning improvements based on their impact on growth and user acquisition.

## Must Fix Before Release
*None.* The brand positioning in the README is exceptionally strong.

## Should Fix Soon (v1.x)
1. **Marketing Landing Page:** Create a static website for the root domain (`canihavesex.today`) that serves as a high-converting landing page. It should feature the tagline prominently and offer two clear paths: "Self-Host (Free)" and "Hosted Cloud (Paid/Beta)".
2. **High-Quality Assets:** Generate professional screenshots and an Open Graph preview image (1200x630) that showcases the clean, dark-mode UI alongside the provocative domain name. This is critical for Twitter, Reddit, and Hacker News sharing.
3. **PWA Manifest Short Name:** Ensure the `manifest.json` uses a discreet short name (e.g., "Today") so users aren't deterred from adding it to their mobile home screens due to the explicit domain name.

## Can Wait Until After v1.0 (v2.0+)
1. **Founder's Manifesto:** Publish a blog post or manifesto detailing *why* this product was built (e.g., the rejection of corporate data harvesting in reproductive health). This will fuel organic growth in privacy communities.
2. **Press Kit:** Develop a simple press kit for journalists covering privacy tech or open-source health software.
